## 1. 目标与决策

把第二轮发现的问题修掉：恢复活动列表草稿「编辑」入口、AI 助理消息乐观渲染、放松生成类 prompt 的过度边界并补足后台助理 / 飞书 Bot 的输入上下文、按 Pine 原型还原基础控件样式并补齐移动端响应式。全程 TDD，先改/补测试再实现。

已确认的规划决策：

- **切分策略**：按测试 spec 1.3 推荐顺序——先两个前端交互（B/A），再 AI 提示词与上下文（C），最后 UI（D）。共 7 个 task，按序执行。
- **飞书 Bot 报名名单**：采纳「小球队默认公开报名名单」。Bot 可回答「这次有哪些人会来」（列出已反馈"去"的队员姓名），system prompt 追加公开声明，user prompt 注入去名单。
- **移动端范围**：核心三件套（侧栏→底部 Tab、活动表格→卡片、中间态横向滚动）+ UI 文档 D 节全量建议（详情 Banner 纵向化、出勤统计卡片两列、出勤名单单列、表单单列+按钮全宽、AI 聊天容器移动端透明/全高）。
- **AI 气泡机器人（UI 问题 4）**：现有 `RobotAvatar.tsx`（香槟金 peeking robot SVG，含悬停上浮）已满足设计意图，**视为已完成，本轮不改**。
- **复盘公开约束**：§3.3 新复盘 prompt 删除了"面向全员公开/不得输出复盘原文/队长备注原文"。复盘摘要全员可见，实现时保留 `memberLine(m,false)`（不喂备注）并在复盘 system prompt 补一句最小公开约束，其余按文档。

## 2. 源码核实结论（与测试 spec 的差异）

逐条对照源码后，spec 的现象与 repo 相对路径定位**全部准确**。需在实现时注意：

- **UI 问题 4（机器人）文档现状描述过时**：spec 写"当前是静态卡通机器人图片/Emoji"，但代码已有 `web-admin/src/components/RobotAvatar.tsx`（香槟金 peeking robot SVG，44×40，带耳朵/手/微笑），且已在 `Assistant.tsx` 气泡左上角使用——应是上一次「全局落地 Pine 设计系统」提交已替换。本轮不改（见决策）。
- **新助理 system prompt 会打破现有测试**：`server/test/assistant.test.ts:24-32` 断言旧措辞「当前判断 / 判断依据 / 只引用系统真实数据」。换成 §4.3 新 prompt 后这些断言失效，必须更新为新契约。
- **现有 prompt 断言在新 prompt 下仍成立（已逐条验证）**：`scenarios.test.ts`（"只返回 JSON 对象"、"不得把…排入首发或替补"、"活动类型：比赛/训练"、"如果无人反馈"、"记得带护具"、不含 captainNote、initial 不含"AI复盘总结"）、`memberBot.test.ts`（不泄露 captainNote、摘要优先 aiSummary）均不受影响——前提是 memberBot 实现保持「摘要恒优先 aiSummary、绝不注入 captainNote」。
- **`.doc-h` 竖杠 bug 确认**：`index.css:96-111` 现用 `margin-left:-11px; padding-left:11px`（正是 UI 文档警告的负 margin 反模式，竖杠被推到段落左边缘左侧 11px），竖杠 `border-radius:999px`（应 2px）；字体 `text-base font-bold` = 14px/700（Pine token `base=14px`）**已达标**。修复 = 删负 margin + 改圆角。
- **radio/checkbox 确认为原生**：`ActivityForm.tsx` 用 `<input type=radio/checkbox className="accent-brand">`，需替换为自定义控件。
- **移动端确认缺失**：`Layout.tsx` 固定 `w-[224px]` 侧栏、无断点、无底部 Tab；`Activities.tsx` 表格已有 `overflow-x-auto + min-w-[940px]`（中间态横向滚动已部分具备），无卡片列表。
- **测试基建齐全**：后端 `scenarios.test.ts / assistant.test.ts / memberBot.test.ts / aiContext.test.ts` 可用 fake LLM 捕获 `(system, user)`；前端 `web-admin/test/*.tsx` 用 vitest + testing-library 覆盖各页。
- **数据模型可支撑新上下文**：`TeamSettings`（defaultLocation/trainingRules/matchRules）、`Activity`（status/startTime/durationMinutes/theme/notes/summary）、`ActivityParticipant`（attendanceResponse∈{going,not_going,no_response}、actualAttendance∈{present,…}）、`Member`（primaryPosition∈{tekong=发球手,feeder=二传手,striker=攻球手}）。位置/水平中文 label 用 `@teampilot/shared` 的 `positionLabel / levelLabel`。
